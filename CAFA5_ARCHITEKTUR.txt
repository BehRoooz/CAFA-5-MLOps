Training Pipeline

Raw Data → Embeddings → Training → MLflow Registry

────────────────────────────────────────────────────────────


The system is structured into multiple layers.

At the top, we have a Streamlit UI for user interaction.

Requests are routed through an NGINX gateway, which acts as a unified entry point and provides authentication for administrative endpoints.

The core inference pipeline consists of two services:
an embedding API using ESM2 for feature extraction,
and a prediction API that loads the current model from MLflow.

The MLflow registry handles model versioning and deployment.

In parallel, the system is monitored using Prometheus and Grafana.

Finally, the training pipeline processes raw protein data, generates embeddings,
trains the model, and registers it in MLflow.
